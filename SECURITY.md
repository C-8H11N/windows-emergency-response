# 安全策略

## 🔒 安全边界

本工具的设计原则：

### ✅ 默认行为

- **纯本地分析** - 所有扫描和分析在本地完成
- **不保存历史数据** - 无内置数据库，重启即清零
- **不外发数据** - 威胁情报默认关闭，不向任何第三方发送数据
- **只读操作** - 不修改注册表、不删除文件、不终止进程（需用户显式确认）
- **不读取凭据** - 不读取 SAM、LSASS 内存、浏览器密码等敏感凭据

### ⚠️ 可选功能

威胁情报功能需要用户**显式开启**：

1. 设置环境变量 `WIN_ER_ENABLE_EXTERNAL_INTEL=1`
2. 设置 API Key 环境变量
3. 在前端设置页手动点击"真实 API"按钮

启用后，仅查询用户**显式选择**的指标（IP/哈希/域名），不会批量上传。

## 🐛 安全漏洞报告

如果你发现了本工具的安全漏洞，请**不要公开提交 Issue**，而是通过以下方式私下报告：

### 报告方式

- **GitHub Security Advisories**（推荐）：前往 [Security tab](https://github.com/yourusername/win-er/security)
- **邮件**：发送详细信息至维护者邮箱

### 报告内容

请包含以下信息：

1. 漏洞类型（例如：RCE、文件读取、XSS、CSRF）
2. 受影响版本
3. 复现步骤或 POC
4. 潜在影响范围
5. 你的联系方式（可选）

### 处理流程

1. **确认** - 24 小时内确认收到报告
2. **评估** - 评估漏洞严重程度和影响范围
3. **修复** - 开发补丁并测试
4. **发布** - 发布安全更新并公开致谢

## 🛡️ 安全使用建议

### 运行环境

1. ✅ 建议在**隔离环境**中测试
2. ✅ 仅在**已授权的系统**上运行
3. ✅ 重要系统建议先做**快照备份**
4. ✅ 关注 [Releases](https://github.com/yourusername/win-er/releases) 及时更新

### 误报与限制

- 本工具为**启发式分析**，存在误报可能
- 所有发现项请**人工复核**后再处置
- 工具结果仅供参考，不能替代专业取证分析
